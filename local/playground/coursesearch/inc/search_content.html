--> New
Column sort in the course list
The table columns can be sorteed via the drop-down menu in the search are or by clicking on the column title.
Course data
The course data is arbitrary.
Course links
The course links link to hard coded course ids. That means the link may open a course or open an error page if a course with the hard coded id does not exisits.
Selection behaviour
Only courses are shown, that fullfill all selectors.
--> Changed
»Toggle search criteria selection« button
All search criteria are listed. The new selection is saved when the display is changed back to the results page. In this click dummy the list of shown courses is not changed when the the search criteria are changed. In the final version the list will be updated related to the preselected tags.
In this click dummy the preselected tags are saved in a browser cookie.
Display options
Sort and visibility aspects of the course list users select also in the left search area. The criteria are also lsited as items above the course list. This way users have a consistent interface with all selected criteria in one place.
Display option »Show the course tags«
The course tags are hidden in the list view and not present in the cards view. They add a lot of information and reduce the clarity of the list. To be able to check the course tags as an important aspect of the search the tags visibility can be toggled on/off.
Date search
The dates are always shown, independend of the selected course types.
Text on cards
The course summary is truncated on the course cards.
Cards on small screens
In the click dummy always three cards are shown in one row. The number may change depending on the screen width.
